# [프로젝트 요약] 구매 프로세스 개선 프로젝트 – 성과 요약

## 1. 프로젝트 개요

목표:
- 승인 프로세스 단순화
- 가격 오류 감소
- MM–FI 통합 안정성 확보

범위:
- PR → PO → GR → IR → 회계 전표
- 승인 전략
- 가격결정
- 계정결정(OBYC)

---

## 2. 주요 개선 내용

### 승인 구조 개선
- 금액 기준 Flexible Workflow 도입
- 10,000 이하 자동 승인
- 승인 단계 평균 30% 단축

### 가격 통제 강화
- Info Record / Condition 유효기간 통제
- 승인 후 가격 변경 시 재승인 로직 적용
- 가격 오류 발생률 감소

### 계정결정 안정화
- OBYC 매핑 재정의
- Valuation Class 검증 로직 강화
- 회계 전표 오류 감소

### GR/IR 관리 체계화
- 월별 잔액 점검 프로세스 도입
- 장기 미정리 잔액 감소

---

## 3. MM–FI 통합 관점 성과

- Movement Type 기반 자동 분개 검증
- BSX / WRX / PRD 흐름 명확화
- Standard vs Moving Average 차이 분석 완료
- 회계 영향 설명 가능 수준 확보

---

## 4. 리스크 대응 전략

- 승인 지연 시 Workflow 로그 점검
- 가격 미결정 시 Pricing Procedure 추적
- 계정 오류 시 OBYC Transaction Key 분석
- 문서 변경 시 Change History 추적

---

## 5. 정량적 기대 효과 (시뮬레이션 기준)

- 승인 리드타임 30% 단축
- 가격 오류 50% 감소
- GR/IR 잔액 관리 개선
- 구매 KPI 모니터링 체계 구축

---

## 6. 컨설턴트 관점 학습 성과

- End-to-End 프로세스 설명 가능
- Customizing 설정 단계 설명 가능
- 오류 트러블슈팅 구조화 가능
- S/4HANA 전환 영향 설명 가능

---

## 7. 결론

본 프로젝트는 단순 기능 구현이 아닌  
프로세스, 설정, 회계, 기술을 연결하여  
통합적 관점의 MM 컨설턴트 역량 확보를 목표로 수행되었다.
