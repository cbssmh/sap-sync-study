# [프로젝트 시뮬레이션] 구매 프로세스 개선 프로젝트 – 요구사항 정의서

## 1. 프로젝트 개요

프로젝트명:
- 구매 프로세스 표준화 및 승인 자동화

목표:
- 승인 단계 단순화
- 가격 오류 감소
- 구매 리드타임 20% 단축

---

## 2. As-Is 프로세스 문제점

1) 금액 기준 승인 단계 과다
2) 가격 Condition Record 누락 빈번
3) GR/IR 잔액 장기 미정리
4) 승인 후 무단 가격 변경 사례 존재

---

## 3. To-Be 프로세스 방향

1) Flexible Workflow 도입
2) 금액 10,000 이하 자동 승인
3) Source List 필수화
4) GR/IR 월별 정리 프로세스 도입
5) Changeability Indicator 재설계

---

## 4. 기능 요구사항

### 4.1 승인 프로세스
- 금액 10,000 이하: 자동 승인
- 10,000~50,000: 팀장 승인
- 50,000 이상: 부서장 2단계 승인

### 4.2 가격 통제
- PO 생성 시 Info Record 필수 체크
- 유효기간 만료 시 오류 메시지 표시

### 4.3 계정결정 검증
- 신규 자재 생성 시 Valuation Class 자동 체크

---

## 5. 비기능 요구사항

- 승인 처리 시간 24시간 이내
- GR/IR 잔액 30일 초과 금지
- 리포트 자동 모니터링 구축

---

## 6. 기대 효과

- 승인 단계 30% 감소
- 가격 오류 50% 감소
- 재고 회전율 개선
- 감사 대응 리스크 감소

---

## 7. 프로젝트 범위

In Scope:
- MM Purchasing
- 승인 프로세스
- 계정결정

Out of Scope:
- SD 판매 프로세스
- 생산 PP 모듈

---

## 8. 컨설턴트 역할

- 요구사항 수집
- To-Be 설계
- Customizing 설정
- 테스트 시나리오 작성
- 사용자 교육
