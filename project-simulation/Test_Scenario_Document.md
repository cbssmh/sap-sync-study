# [프로젝트 시뮬레이션] Test Scenario 문서

## 1. 테스트 개요

목적:
- To-Be 프로세스 정상 동작 검증
- 승인 / 가격 / 계정결정 / 회계 연계 확인

범위:
- PR → PO → GR → IR → 회계 전표

---

## 2. 단위 테스트 (Unit Test)

### TC-01 : 자동 승인 검증

조건:
- PR 금액 8,000

예상 결과:
- 자동 승인 처리
- PO 생성 가능

---

### TC-02 : 2단계 승인 검증

조건:
- PO 금액 60,000

예상 결과:
- 팀장 승인 → 부서장 승인
- 승인 완료 후 출력 가능

---

## 3. 가격 결정 테스트

### TC-03 : Info Record 가격 적용

조건:
- Vendor + Material Info Record 존재

예상 결과:
- PB00 자동 적용
- 총 금액 정상 계산

---

### TC-04 : 가격 유효기간 만료

조건:
- Condition Record 만료

예상 결과:
- 오류 메시지 발생
- PO 저장 불가

---

## 4. 계정결정 테스트

### TC-05 : GR 회계 전표 생성

조건:
- Movement Type 101
- Valuation Class 3000

예상 결과:
- BSX 차변
- WRX 대변

---

### TC-06 : Standard Price 차이 발생

조건:
- 구매가 > 표준가

예상 결과:
- PRD 계정 차이 발생

---

## 5. 송장 검증 테스트

### TC-07 : 3-Way Match 정상

조건:
- PO/GR/IR 금액 일치

예상 결과:
- 블로킹 없음
- GR/IR 상계 완료

---

### TC-08 : 허용 오차 초과

조건:
- 금액 차이 5%

예상 결과:
- Invoice Block 발생
- MRBR에서 확인 가능

---

## 6. 통합 테스트 시나리오

TC-09 : End-to-End 전체 흐름

PR 생성 → 승인 → PO 생성 → GR → IR → 회계 전표 확인

검증 포인트:
- 문서 흐름 정상 연결
- Accounting Document 자동 생성
- GR/IR 잔액 0

---

## 7. 테스트 완료 기준

- 주요 시나리오 100% 통과
- 승인 지연 없음
- 가격/계정 오류 없음
- 회계 전표 정확성 검증 완료
