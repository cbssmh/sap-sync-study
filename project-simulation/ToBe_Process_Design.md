# [프로젝트 시뮬레이션] To-Be 구매 프로세스 설계서

## 1. 설계 목표

- 승인 단계 단순화
- 가격 오류 최소화
- 구매 리드타임 단축
- GR/IR 잔액 관리 체계화

---

## 2. 전체 To-Be 흐름

요청 → 자동 소싱 → 승인 → 발주 → 입고 → 송장 → 회계 정리

---

## 3. 단계별 설계

### 3.1 Purchase Requisition (PR)

- MRP 실행 시 자동 PR 생성
- Source List 필수 적용
- 금액 10,000 이하 자동 승인
- 10,000 초과 시 Flexible Workflow 적용

개선 포인트:
- 수동 공급업체 선택 최소화
- 승인 대기 시간 단축

---

### 3.2 Purchase Order (PO)

- Info Record 기반 가격 자동 제안
- 계약 존재 시 계약 가격 우선 적용
- 승인 완료 후에만 출력 허용

통제 포인트:
- 승인 후 가격 변경 시 재승인 필수
- Change Log 활성화

---

### 3.3 Goods Receipt (GR)

- 입고 시 Movement Type 101 표준화
- Storage Location 사전 정의
- 자동 계정결정(OBYC) 검증 완료 상태

재고 통제:
- 실사 주기 월 1회
- 30일 초과 미입고 PO 자동 리포트

---

### 3.4 Invoice Receipt (IR)

- 3-Way Match 필수 적용
- 허용 오차 범위 ±3%
- 초과 시 자동 블로킹

관리 방안:
- MRBR 주간 점검
- GR/IR 월말 정리 프로세스 운영

---

## 4. 통제 설계 요약

| 영역 | 통제 방식 |
|------|------------|
| 승인 | Flexible Workflow |
| 가격 | Condition Record 유효기간 관리 |
| 계정 | OBYC 사전 검증 |
| 재고 | 월간 실사 |
| 회계 | GR/IR 정기 정리 |

---

## 5. 기대 효과

- 승인 리드타임 30% 단축
- 가격 오류 50% 감소
- GR/IR 잔액 감소
- 재고 정확도 개선

---

## 6. 컨설턴트 역할

- 프로세스 재설계
- Customizing 반영
- 테스트 시나리오 작성
- 사용자 교육 및 안정화 지원
