# SAP MM 재고 이전 / 이동 (Stock Transfer) 정리

## 1. 재고 이전이란?
- 자재를 한 장소에서 다른 장소로 이동시키는 프로세스
- 회사 내 재고 위치 변경이 목적
- 구매/판매가 아닌 내부 이동

---

## 2. 재고 이전 유형

### Storage Location 간 이동
- 같은 Plant 내에서 창고만 이동
- 수량만 이동, 금액 영향 없음

대표 Movement Type:
- 311 : Storage Location → Storage Location

---

### Plant 간 이동
- 서로 다른 Plant 간 재고 이동
- 재고 평가/회계 영향 발생 가능

대표 Movement Type:
- 301 : Plant → Plant (회사코드 동일)
- 303 / 305 : 2단계 이동 (출고 → 입고)

---

## 3. 1단계 vs 2단계 이동

### 1단계 이동
- 한 번의 전기로 즉시 이동 완료
- 단순, 빠름

### 2단계 이동
- 출고 후 입고 처리
- 운송 중 재고(In-transit) 관리 가능
- 장거리/시간차 이동에 적합

---

## 4. 주요 T-Code
- MIGO : 재고 이동 처리
- MB51 : 자재 이동 이력 조회

---

## 5. 실무 포인트
- 이동유형 선택 오류 빈번
- Plant 간 이동 시 평가 방식 영향 확인 필요
- In-transit 재고 여부 확인 중요
- 컨설턴트는 “어디서 → 어디로” 이동했는지 먼저 파악
