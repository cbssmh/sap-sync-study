# SAP MM Invoice Verification 심화 정리

## 1. Invoice Verification이란?
- 공급업체 송장을 검증하고 회계 전표로 전기하는 프로세스
- PO / GR 기준으로 수량·금액 일치 여부 확인
- MM과 FI를 직접 연결

---

## 2. 기본 처리 흐름
1) PO 생성
2) GR 처리 (선택적)
3) MIRO로 송장 입력
4) 검증 통과 시 회계 전표 생성
5) 지급 대기 상태로 관리

---

## 3. 검증 기준 (3-Way Match)
- PO : 주문 수량/가격
- GR : 실제 입고 수량
- IR : 송장 수량/금액

→ 세 값의 일치 여부가 핵심

---

## 4. 차이 발생 유형
- 가격 차이
- 수량 차이
- 세금/운임 차이

처리 방식:
- 허용 한계 내 → 자동 전기
- 초과 시 → 블로킹(Block)

---

## 5. Invoice Block
- 송장 지급을 일시 중단
- 원인 해결 후 해제 필요

주요 원인:
- 금액 초과
- 수량 불일치
- 가격 조건 오류

---

## 6. 주요 T-Code
- MIRO : 송장 입력
- MIR4 : 송장 조회
- MRBR : 송장 블록 해제

---

## 7. 실무 포인트
- GR 없는 송장 처리 여부 정책 확인 필요
- 블로킹 사유부터 정확히 파악
- GR/IR 잔액과 함께 분석해야 함
- 컨설턴트는 “왜 지급이 안 되는지” 설명 가능해야 함
