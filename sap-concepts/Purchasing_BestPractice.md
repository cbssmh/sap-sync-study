# SAP MM 구매 자동화 / Best Practice 정리

## 1. 구매 자동화란?
- 반복적인 구매 업무를 시스템이 자동 처리하도록 설계
- 수작업 최소화, 오류 감소, 처리 속도 향상이 목적

---

## 2. 구매 자동화 적용 영역

### PR 자동 생성
- MRP 실행 결과로 PR 자동 생성
- 재고 부족 시 자동 대응

---

### Source Determination
- Source List / Info Record / Quota Arrangement 활용
- 공급업체 자동 제안

---

### 가격 자동 결정
- Condition Technique 기반 가격 자동 계산
- 계약/Info Record 가격 우선 적용

---

### 승인 자동화
- Release Strategy / Workflow 활용
- 금액/조건별 자동 승인 경로

---

## 3. 자동화에 필요한 핵심 설정
- 정확한 Master Data
- Source List 유지
- Pricing Condition 관리
- 승인 조건 명확화

---

## 4. 구매 Best Practice
- 표준 구매 프로세스 유지
- 예외 케이스 최소화
- 승인 단계 단순화
- 리포트 기반 사전 모니터링

---

## 5. 자동화 시 주의사항
- 마스터 데이터 오류가 자동으로 확산됨
- 테스트 없이 자동화 적용 위험
- 현업 승인/통제 요건 반영 필수

---

## 6. 실무 포인트
- 자동화는 단계적으로 적용
- KPI로 효과 측정
- 예외 처리 프로세스 반드시 정의
- 컨설턴트는 “왜 자동화했는지” 설명 가능해야 함
