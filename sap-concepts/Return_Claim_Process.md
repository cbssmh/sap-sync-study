# SAP MM 반품 / 클레임 프로세스 정리

## 1. 반품/클레임이란?
- 입고된 자재에 대해 불량, 수량 오류, 사양 불일치 등이 발생했을 때
  공급업체에 반품하거나 보상(클레임) 처리하는 프로세스

---

## 2. 반품이 필요한 대표 상황
- 불량 자재 납품
- 과다/과소 납품
- 사양 불일치
- 운송 중 파손

---

## 3. 반품 처리 기본 흐름

### 1) 반품 PO 생성
- 기존 PO 참조
- Return Item으로 설정
- 수량은 음수 개념

주요 T-Code:
- ME21N (Return PO)

---

### 2) 반품 출고 처리
- 공급업체로 자재 반출
- 재고 감소
- 회계 전표 생성

주요 Movement Type:
- 122 : 반품 출고

주요 T-Code:
- MIGO

---

### 3) 크레딧 메모(환불)
- 공급업체에서 Credit Memo 수령
- 송장 금액 차감 처리

주요 T-Code:
- MIRO (Credit Memo)

---

## 4. 클레임 처리 방식
- 수량 반품
- 가격 할인
- 차액 환불
- 대체 자재 재납품

---

## 5. 문서 흐름
- PO → Return PO → Material Document → Accounting Document → Credit Memo

---

## 6. 실무 포인트
- 반품 PO 여부 반드시 확인
- Movement Type 혼동 빈번 (122 vs 102)
- 금액 클레임은 GR/IR 잔액과 연계 확인
- 컨설턴트는 “물류 반품인지, 금액 클레임인지” 구분 가능해야 함
