# ABAP BDC(Batch Data Communication) 기본 정리

## 1. BDC란?
- SAP 화면(Transaction)을 자동으로 실행해 데이터를 입력하는 방식
- 대량 데이터 업로드, 반복 업무 자동화에 사용
- 기존 표준 T-Code를 그대로 사용

---

## 2. BDC 사용 목적
- 엑셀/외부 파일 데이터를 SAP에 일괄 입력
- 수작업 입력 오류 감소
- 초기 데이터 마이그레이션에 활용

---

## 3. BDC 처리 방식

### Session 방식
- BDC 세션 생성 후 실행
- 오류 발생 시 재처리 가능
- 대량 데이터에 적합

### Call Transaction 방식
- 즉시 실행
- 실시간 처리
- 오류 처리 로직 직접 구현 필요

---

## 4. 기본 구성 요소
- BDCDATA : 화면 필드 정보 저장 구조
- OKCODE : 화면 버튼/동작 지정
- Program / Screen 번호 기반 처리

---

## 5. BDC 기본 흐름
1) 입력 데이터 준비
2) 화면 순서 분석 (SHDB)
3) BDCDATA 구성
4) BDC 실행
5) 결과 로그 확인

---

## 6. 실무 포인트
- 화면 변경 시 BDC 오류 발생 가능
- 성능/유지보수 측면에서 LSMW/인터페이스 고려
- 컨설턴트는 “왜 BDC를 쓰는지” 설명 가능해야 함
