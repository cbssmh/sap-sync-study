# ABAP Debugging 기초 정리

## 1) Debugger란?
- ABAP 프로그램 실행 과정에서 변수 값, 내부 테이블, 흐름을 확인하는 도구.
- 오류 분석, 로직 파악에 필수.

## 2) Breakpoint 설정
- 특정 지점에서 프로그램을 멈추게 함.

방법 1: 코드에 직접 명령어 입력  
BREAK-POINT.

방법 2: 에디터에서 줄 번호 클릭 → Breakpoint 생성.


## 3) 변수 값 확인
- Debugger 창 좌측 하단 "Variables" 영역에서 현재 변수 값 확인 가능.
- 내부 테이블은 Contents로 전체 행 조회 가능.


## 4) Single Step 실행 방식
- F5 : 한 단계씩 실행(다음 명령어)  
- F6 : 함수 내부로 진입하지 않고 실행  
- F7 : 현재 루틴(LOOP, FORM 등) 빠져나가기  
- F8 : 다음 Breakpoint까지 실행  


## 5) Watchpoint
- 특정 변수 값이 변화할 때 자동으로 멈추는 기능.

예:  
변수 lv_total 값이 변할 때 멈추게 설정  
→ 문제 원인 추적에 매우 유용.


## 6) Debugging이 왜 중요한가?
- SELECT가 어떤 값을 가져오는지 확인 가능  
- LOOP 안에서 값이 어떻게 변하는지 추적  
- MM 오류(예: PO/GR/IV 데이터 불일치) 원인을 빠르게 찾을 수 있음  
- 개발자와 협업할 때 “문제 지점”을 명확히 전달 가능
